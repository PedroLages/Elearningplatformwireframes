## Key Insights and Learnings

1. **Fast epics sacrifice documentation.** 5 stories in 3 days left no room for Implementation Notes in S03/S04. Speed and documentation completeness are in tension — must explicitly allocate time for notes.

2. **E2E tests need AC-level assertion mapping.** "Test passes" ≠ "AC verified." Each AC with specific metrics (contrast ratio, pixel size, animation duration) needs an explicit assertion. Behavioral smoke tests are insufficient.

3. **Deferred debt escalation is necessary.** Items appearing in 3+ retros without resolution need a forcing function. New rule: after 3rd retro mention, item becomes first task of next epic or is explicitly dropped.

4. **Dashboard performance budget matters now.** Epic 4 added multiple dashboard-facing stores. Epic 5 adds streak widgets, goal progress, and calendar — all hitting the same stores. Redundant calls will compound.

5. **Backward-compatible component evolution works.** S02's Progress enhancement proves: optional props, value normalization, and conditional rendering enable adding features without breaking existing callsites.

6. **`mode="display"` pattern for nested interactives.** When a component can be interactive (button) or read-only (span), a `mode` prop controls rendering. Established in S01, applicable anywhere an indicator lives inside an interactive container.

7. **Post-review fix velocity improved.** S02 went from 2 blockers + 7 high findings to fully fixed in one commit cycle. The review → fix → verify loop is maturing.
