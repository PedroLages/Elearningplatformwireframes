## Successes and Strengths

1. **Parse-once pattern (`getStreakSnapshot()`) eliminated redundant localStorage reads.** Single parse derives current streak, longest streak, and 90-day activity array. Became the architectural template for the epic.

2. **DST-safe date handling established as team standard.** `toLocaleDateString('sv-SE')` for timezone-safe date strings, `parseLocalDate()` with `Math.round()` on day diffs. Prevented real user-facing bugs near midnight and DST transitions.

3. **CustomEvent bus for cross-component reactivity.** `study-log-updated`, `streak-pause-updated`, `study-reminders-updated` — lightweight alternative to prop drilling or state management overhaul.

4. **Story documentation dramatically improved.** All 6 stories have implementation notes, challenges, and lessons learned. Epic 4 commitment honored.

5. **Pure-function architecture for computations (S03).** Progress calculations as pure functions enabled surgical review fixes — zero React component changes needed.

6. **Defense-in-depth validation (S02).** Freeze days validated on both write (`setFreezeDays` truncates to 3) and read (`getFreezeDays` re-validates). Protects against corrupted localStorage.

7. **Fast post-review fix cycles.** Despite every story needing 2-3 review rounds, fix velocity was consistently fast. Team internalized the review-fix-verify loop.

8. **Zero production incidents, 100% delivery.** All 6 stories shipped with all review gates passed.

## Key Insights and Learnings

1. **Documented lessons without enforcement mechanisms repeat.** The worktree dev server collision was documented in S03's lessons learned but repeated in S05. Writing lessons in story files that nobody reads before the next story is equivalent to a verbal agreement. Needs automation (hook, script, or checklist).

2. **Commit-before-review is trivially simple but keeps getting missed.** A 10-second `git add && git commit` eliminates an entire category of blocker findings. Must become a pre-review gate.

3. **`toLocaleDateString('sv-SE')` is the timezone-safe date pattern.** `toISOString().split('T')[0]` returns UTC, not local — near-midnight users in western timezones get wrong dates. Established as team standard.

4. **`addInitScript` runs on every page load including reloads.** Not just once per test. Using `localStorage.clear()` inside it wipes seeded test data on `page.reload()`. Fix: sessionStorage flag to run setup only once per test.

5. **Vitest mock constructors need class-based mocks.** `vi.fn()` cannot be used with `new`. Use `class MockNotification` with `vi.stubGlobal()`.

6. **Tailwind `-700` variants minimum for status text.** `-600` variants fail WCAG AA on white backgrounds (green-600: 3.30:1, amber-600: 3.19:1). Use `-700` minimum (4.87:1 and 4.55:1).

7. **Canvas-confetti colors are an accepted theme bypass.** Confetti hex colors can't use CSS variables. Document in code comments rather than fighting it.

8. **Pure-function architecture pays off at review time.** When computation is a pure function, fixing review feedback is surgical — no component rewiring needed.

9. **Defense-in-depth: validate on both write AND read paths.** Even with validated write paths, corrupted localStorage (manual edits, migrations) can inject bad data. Always re-validate on read.

10. **Always use `/start-story` for branch creation.** Ad-hoc branches cause naming mismatches and force-push complications.
