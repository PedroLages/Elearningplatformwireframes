### Critical Learning: Why Some Action Items Stick and Others Don't

**Action items that worked** had tooling or process enforcement:
- AC tech stack audit → built into /start-story planning checklist
- Utility extraction at 2nd usage → developers recognized pattern naturally
- One keyboard handler → simple rule, easy to verify

**Action items that failed** relied on human memory:
- Touch targets → no base component enforcing minimum
- Clean working tree → workflow creates the inverse situation
- Shared test factories → no template or enforced structure

**Central insight: Agreements backed by tooling stick. Verbal commitments requiring discipline without enforcement fail.**

## Challenges and Growth Areas

1. **Touch target violations: 11 out of 15 stories.** Third consecutive retro flagging this issue. Toolbar buttons at 32px, close buttons at 28px, grid cells at 28px, bubble menu at 36px, details toggles at 20px. Team agreement from Epic 2 retro (44px minimum) not honored because no enforcement mechanism exists.

2. **Uncommitted code false-positive blocker: 7 out of 15 stories.** Code review template flags "uncommitted files" as a blocker, but the workflow sequence is implement → review → commit (via /finish-story). This is a workflow design flaw, not a discipline failure. Created unnecessary noise across 7 stories.

3. **Fire-and-forget async handlers: 4+ stories.** `handleDelete`, `handleSave`, and `handleNoteChange` called async functions without try/catch in S06, S07, S09, S10. `handleNoteChange` specifically was flagged in S03, S04, S06, and S09 — same function, same bug, four stories.

4. **Nested interactive elements: 4 stories.** `<Button>` inside `<div role="button">` creates WCAG 4.1.2 violations with two tab stops per card. Appeared in S02, S06, S07, S08 despite being documented as a lesson learned in S06.

5. **Shared test factories never created (3rd retro mention).** Still using inline `makeNote()`, `makeVideo()`, `seedNotes()` per test file. Code review flagged this in S08 and S10.

6. **Theme token cleanup still pending (4th retro mention).** `bg-blue-600` hardcoded across codebase instead of `--primary` CSS variable. Acknowledged as intentionally deferred low-priority debt.

7. **Tiptap `any` casts accumulated.** S13 had 7 `eslint-disable` lines for `any` in Tiptap extension code. Module augmentation was suggested but not implemented.

8. **E2E test coverage depth is inconsistent.** Multiple stories had "partial" AC coverage in testing reviews — specific metric ACs (latency, format validation) and error paths were undertested.

## Key Insights and Learnings

1. **Build systems, not habits.** Agreements backed by tooling enforcement work (AC tech stack audit, utility extraction trigger). Agreements requiring human memory and discipline fail (touch targets, clean working tree). Every future retro action item must have a technical artifact that enforces it.

2. **The "uncommitted files" blocker is a workflow design flaw.** The review evaluates code quality, not git status. Fix the code review template to remove this false-positive criterion. The implement → review → commit sequence is correct for the workflow.

3. **Progressive architecture enables ambitious epics.** Epic 3 scaled from 7 to 15 stories because each built on a solid foundation. The data layer (S00) → editor (S01) → layout (S02) chain was load-bearing architecture that unlocked everything after it.

4. **Component decomposition at complexity boundaries is now a pattern.** When a component exceeds 300 lines or 5 features, decompose into directory with barrel exports. Established in S13, validated in S14.

5. **Tiptap extension storage beats global events for cross-component communication.** S09 replaced `window.dispatchEvent(CustomEvent)` with `editor.storage.frameCapture.onSeek` — scoped to editor instance, no stale closures, no global coupling.

6. **Custom protocol registration pattern is reusable.** Register in StarterKit `link.protocols`, extend `rehype-sanitize` schema, override `urlTransform`. Works for `video://`, `screenshot://`, and any future protocol.

7. **Explicit UI beats magic extraction.** S04's tag management via dedicated TagEditor (replacing automatic hashtag parsing) eliminated false positives and gave users clear control. Applies to any feature where "automatic detection" could misfire.

8. **Async event handlers must ALWAYS have error handling.** Four stories with fire-and-forget patterns. Pattern: wrap all `onClick` async handlers in try/catch with user-visible error feedback (toast on failure).

9. **E2E test assertions must target AC metrics, not just behavior.** Acceptance criteria with specific numbers (< 100ms, 44px, highlighted keywords) need explicit assertions. Behavioral smoke tests are insufficient.

10. **Extract shared utilities at the second usage.** When a utility appears in 2 files, extract before the third. formatTimestamp, ReadOnlyContent, stripHtml/truncateText all validated this pattern.
