## Key Insights and Learnings

1. **Lessons must be actively read at story start.** Documenting lessons after a story is insufficient. The SM must surface the last 3 stories' "Challenges and Lessons Learned" sections as an active checklist before finalizing the implementation plan for the next story. This is a workflow change, not a skill gap.

2. **AC tech stack validation is a planning responsibility.** When an AC specifies a technology (IndexedDB, localStorage, react-player), the SM must verify it matches the established tech stack during /start-story. Mismatches resolved at planning time cost minutes; discovered at code review cost full story cycles.

3. **Pre-review clean working tree is a quality gate.** `git status` must show no modified files before requesting review. Working-tree fixes don't count as shipped ‚Äî only committed code is reviewable code.

4. **Design system defaults prevent recurring failures.** Touch target violations recur because each component starts from zero. The fix is a design system default: `min-h-[44px] min-w-[44px]` on all icon-only interactive elements, enforced by convention rather than per-component review.

5. **Prop completeness requires full interface audit.** When a parent renders a child after the current story adds a new prop, audit ALL props on the child's interface ‚Äî not just the ones this story added. The captions prop was missed twice across two different stories.

6. **One keyboard handler per shortcut.** Global keyboard shortcuts must live in exactly one place ‚Äî either a window listener or a React synthetic event, never both. Having both causes double-toggle (S09: T key) and event propagation conflicts (S07: ? key).

7. **Atomic commits: data, fixtures, and tests together.** Test fixtures, sample data, and the tests that depend on them must be committed atomically. Separate commits create states where tests reference uncommitted data, causing false failures (S08 B1: E2E tests failed because course data was uncommitted).

8. **IntersectionObserver requires ancestor scroll container.** IntersectionObserver only fires correctly when scrolling happens on an ancestor of the observed element. The mini-player in S09 initially failed because the scroll event was on a sibling container. Always trace the actual scroll container before wiring intersection logic.

9. **CSS `inset-0` is four properties.** When switching an element from `absolute inset-0` to `position: fixed`, add `top-auto left-auto` explicitly ‚Äî `inset-0` sets all four sides; overriding `bottom` and `right` alone leaves `top: 0; left: 0` active, creating an invisible full-viewport click trap.

10. **Effect deps that read and write the same value are red flags.** Any `useEffect` that both reads and writes a value it depends on (like `remaining` in AutoAdvanceCountdown) will tear down and recreate the effect every tick. Store the callback in a ref; use a single stable `setInterval`.
