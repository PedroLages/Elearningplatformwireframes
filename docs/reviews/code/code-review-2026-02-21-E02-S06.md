# Code Review: E02-S06 — Video Player UX Fixes & Accessibility

**Date**: 2026-02-21
**Story**: E02-S06
**Reviewer**: Adversarial Code Review Agent

## What Works Well

1. **Speed menu keyboard navigation is well-engineered.** Document-level `keydown` with capture phase solves Safari focus issues, and `flushSync` before focus is thoughtful.
2. **Touch/mouse event deduplication is correct.** The `touchActiveRef` pattern with 500ms cooldown prevents double-fire.
3. **AutoAdvanceCountdown fixes the S05 interval bug.** Uses stable interval with functional updater and ref pattern for `onAdvance`.
4. **Scrollbar theming is clean.** CSS custom properties with both standards-based and WebKit fallback, proper dark mode scoping.

## Findings

### Blockers (2)

**B1 — Focus ring uses `focus:` instead of `focus-visible:`** (`VideoPlayer.tsx:489`)
Container uses `focus:outline` which shows focus ring on mouse clicks. AC2 specifies "When the user tabs to the player". Fix: Change all `focus:` to `focus-visible:` on the container.

**B2 — Speed menu has no click-outside-to-close handler** (`VideoPlayer.tsx:651-678`)
Once opened, clicking outside the speed menu does not close it. Standard usability expectation for dropdown menus. Fix: Add a `useEffect` with `document.mousedown` listener when `speedMenuOpen` is true.

### High Priority (4)

**H1 — AutoAdvanceCountdown interval leak past zero** (`AutoAdvanceCountdown.tsx:22-26`)
`setInterval` keeps ticking after `remaining` reaches 0, decrementing to -1, -2, etc. Multiple `onAdvance` calls possible. Fix: Clear interval when reaching zero.

**H2 — `poster` prop not wired** (`LessonPlayer.tsx:186-203`)
`poster` added to `VideoPlayerProps` but `LessonPlayer` never passes it. AC3 says "optional poster attributes" — infrastructure built but unused. Fix: Wire up or document as intentionally deferred.

**H3 — No AC4 (reduced motion) E2E test** (`story-2-6.spec.ts`)
Test file lists AC1, AC2, AC3, AC5 but omits AC4 entirely. No verification of `prefers-reduced-motion` behavior. Fix: Add test using `page.emulateMedia({ reducedMotion: 'reduce' })`.

**H4 — ModuleAccordion stale `openModules` closure** (`ModuleAccordion.tsx:28-34`)
`useEffect` reads `openModules` but it's not in the dependency array `[activeLessonId, modules]`. Can cause module to not re-expand on Next button navigation. Fix: Use functional updater form in `setOpenModules`.

### Medium (4)

**M1** — Mobile volume popover no click-outside-to-close (`VideoPlayer.tsx:616-629`)
**M2** — Center play button uses `h-16 w-16` instead of `size-16` (`VideoPlayer.tsx:539,543`)
**M3** — ModuleAccordion uses string interpolation instead of `cn()` (`ModuleAccordion.tsx:74-78`)
**M4** — ModuleAccordion icons use `h-5 w-5` instead of `size-5` (`ModuleAccordion.tsx:81-83,92-93`)

### Nits (4)

- Redundant `tabIndex={0}` on speed trigger button (`VideoPlayer.tsx:644`)
- LessonPlayer still uses `h-4 w-4` on several icons (out of scope)
- Touch test error-catching pattern less idiomatic than try/catch
- AC3 test doesn't verify `poster` attribute
