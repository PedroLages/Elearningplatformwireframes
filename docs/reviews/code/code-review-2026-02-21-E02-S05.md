## Findings

### Blockers

1. **Unstable countdown timer** — `AutoAdvanceCountdown.tsx:23-34`: `useEffect` has `remaining` in deps, tearing down/recreating interval every second. If `onAdvance` changes identity (it does — inline arrow at LessonPlayer:211), effect re-fires and may call `onAdvance` multiple times. **Fix:** Use a ref for callbacks and a single stable `setInterval`.

2. **Uncontrolled accordion won't re-expand on lesson change** — `ModuleAccordion.tsx:21-26`: `defaultValue` only applies on mount. Navigating from Module A to Module B via Next button won't auto-expand Module B. Breaks AC3 (highlighted lesson in collapsed module is invisible). **Fix:** Switch to controlled `value`/`onValueChange` with `useEffect` to add active module.

### High Priority

3. **Missing prefers-reduced-motion support** — `AutoAdvanceCountdown.tsx`: Task 3.4 explicitly requires this. No motion-related code present. Users with reduced motion should have auto-advance disabled or paused.

4. **Missing E2E test for auto-navigation** — `story-2-5.spec.ts`: AC5 says "auto-advance navigates to the next lesson after countdown" but no test verifies the URL changes after countdown completes.

5. **Auto-advance on re-watch** — `LessonPlayer.tsx:101-114`: `handleVideoEnded` always shows countdown when `nextLesson` exists, even for already-completed videos. Users re-watching material get unexpected auto-advance.

### Medium

6. **String interpolation instead of cn()** — `ModuleAccordion.tsx:63-67`: Use `cn()` from utils per codebase convention.

7. **Redundant handleCancel wrapper** — `AutoAdvanceCountdown.tsx:19-21`: Passthrough useCallback adds nothing. Pass `onCancel` directly.

8. **h-5 w-5 instead of size-5** — `ModuleAccordion.tsx:70-72`: Use Tailwind v4 shorthand.

9. **No-op assertion** — `story-2-5.spec.ts:137`: `expect(count).toBeGreaterThanOrEqual(0)` always passes.

### Nits

10. **Inline callbacks** — `LessonPlayer.tsx:211-215`: Extract `onAdvance`/`onCancel` into `useCallback`.
11. **Dead code** — `LessonList.tsx`: No remaining imports; consider removing.
12. **Fragile selector** — `story-2-5.spec.ts:154`: `a.bg-blue-50` relies on Tailwind classes. Consider `data-active="true"` attribute.
