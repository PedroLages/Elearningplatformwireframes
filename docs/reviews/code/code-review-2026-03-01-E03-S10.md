## Code Review: E03-S10 — Note Export

### What Works Well

1. **Clean three-layer architecture.** The separation of pure conversion functions (`htmlToMarkdown`, `generateFrontmatter`, `sanitizeFilename`), export orchestration (`exportNotes`, `exportSingleNote`), and UI (`ExportNotesDialog`) is well-designed and makes the conversion layer independently testable.

2. **Dynamic JSZip import.** Using `await import('jszip')` inside `exportNotes()` only when ZIP generation is actually needed is a smart bundle-size optimization that avoids bloating the Notes page chunk.

3. **ExportNotesDialog state management.** The dialog properly resets state on close (`handleOpenChange`), wraps the async export in try/catch with a loading spinner, and presents a clear completion summary with failure details. This is the most robust error handling in any recent story.

### Findings

#### Blockers

- **[Recurring] Uncommitted implementation files (confidence: 98)**: The three core implementation files -- `src/lib/noteExport.ts`, `src/app/components/notes/ExportNotesDialog.tsx`, and `src/lib/__tests__/noteExport.test.ts` -- are untracked (`??` in git status). The committed branch contains only the story doc, sprint-status update, E2E spec, package.json change, and diffs to Notes.tsx/CourseNotesTab.tsx that import from the untracked files. **Anyone cloning this branch gets import errors and a broken build.** Fix: `git add` the three untracked files and commit them before merging.

- **[`src/lib/noteExport.ts`:138] (confidence: 92)**: YAML frontmatter injection via unescaped double quotes and newlines. `generateFrontmatter()` interpolates `lessonTitle` and `courseName` directly into double-quoted YAML strings without escaping. A lesson titled `He said "hello"` produces `title: "He said "hello""` which is invalid YAML. Fix: Escape double quotes and newlines in YAML string values.
