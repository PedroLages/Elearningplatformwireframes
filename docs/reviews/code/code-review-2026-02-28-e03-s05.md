# Code Review: E03-S05 — Full-Text Note Search

**Date:** 2026-02-28
**Reviewer:** code-review agent (adversarial)

## What Works Well

1. Clean search architecture — MiniSearch singleton with separate enrichment lookup maps and CRUD lifecycle
2. Incremental index updates already wired into progress operations
3. Custom cmdk filter bypass for note relevance
4. `toSearchableNote()` extraction avoids duplication across index mutation paths

## Findings

### High Priority

**notesOpen deep-link limited to fresh mounts** (confidence: 92)
- `LessonPlayer.tsx:63`: `useState(() => searchParams.get('panel') === 'notes')` only runs on mount. Intra-app navigations reusing the component won't re-open notes panel.
- Primary flow (Overview → search → LessonPlayer) works (fresh mount). Edge case: already on LessonPlayer → search → different lesson → notes panel stays closed.
- Fix: Add `useEffect(() => { if (searchParams.get('panel') === 'notes') setNotesOpen(true) }, [searchParams])`

**Missing keyword highlighting in snippets** (confidence: 85)
- `SearchCommandPalette.tsx:256`: AC requires "highlighted matching keywords" but `truncateSnippet` returns plain text with no visual emphasis.
- Fix: Implement `highlightMatches(text, query)` wrapping terms in `<mark>` elements.

**`buildSearchIndex()` on every render** (confidence: 90)
- `SearchCommandPalette.tsx:197`: Called in render body — rebuilds full course/module/lesson index on every keystroke.
- Fix: `useMemo(() => buildSearchIndex(), [])` or hoist to module scope.

**E2E timestamp test doesn't assert `t=42`** (confidence: 92)
- `story-e03-s05.spec.ts:218-231`: "note with timestamp seeks video" only asserts `panel=notes`, not `t=42`.
- Fix: `expect(page.url()).toContain('t=42')`

**No fuzzy matching unit test** (confidence: 85)
- AC2 fuzzy behavior has zero unit-level coverage. Only tested in E2E.
- Fix: Add `searchNotes('custm')` test asserting results contain "custom".

### Medium Priority

**searchParams dependency in seek effect** (confidence: 78)
- `LessonPlayer.tsx:180-188`: Effect re-runs on any searchParam change. Extract `searchParams.get('t')` as dependency.

**CommandEmpty message timing flicker** (confidence: 75)
- `SearchCommandPalette.tsx:240-243`: Uses `debouncedQuery` for `hasActiveQuery`. During 150ms debounce window, generic message flashes. Use `searchQuery` instead.

**Module-level mutable state in noteSearch** (confidence: 80)
- `noteSearch.ts:15-24`: Singleton `miniSearch` and `courseNameMap`/`lessonTitleMap` leak between test suites. Fragile isolation.

### Nits

- `h-4 w-4` → `size-4` on StickyNote icon (Tailwind v4 shorthand)
- Empty `catch` blocks in `updateInIndex`/`removeFromIndex` swallow all errors
- `truncateSnippet` regex strips `Array<string>` style angle brackets in code snippets

## Verdict

No blockers. 5 high-priority, 3 medium, 3 nit findings.
