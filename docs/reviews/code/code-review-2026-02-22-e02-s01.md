## Findings

### Blockers

- **[CourseCard.tsx:628-641] progress variant missing focus ring**: The `progress` variant `<article>` wrapper has `className="h-full"` â€” no `focus-visible:ring-2`, no `outline-none`, no rounded clipping. Compare with library/overview variant `className="rounded-[24px] focus-visible:ring-2 focus-visible:ring-brand focus-visible:ring-offset-2 outline-none block h-full cursor-default"`. A keyboard user tabbing through My Class will see no focus indicator on progress cards, violating WCAG 2.4.7 (Focus Visible). Fix: Add the same focus-visible classes to the progress variant article at line 634.

- **[CourseCard.tsx:631] progress variant onKeyDown bubbling**: The `onKeyDown` handler does NOT check `e.target === e.currentTarget`. Pressing Enter while focused on the inner "Resume Learning" or "Review Course" Link button fires the article's `onKeyDown` AND the link's native Enter, potentially triggering double navigation. Fix: Add `e.target === e.currentTarget` guard in the `onKeyDown` handler.
