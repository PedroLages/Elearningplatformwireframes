- **`Challenges.tsx:18` (confidence: 92)**: `formatDeadline` uses `new Date(deadline)` — parses date-only strings as UTC, displays wrong date in UTC-negative timezones. Fix: parse with `new Date(y, m-1, d)`.
- **`Challenges.tsx:25` (confidence: 92)**: `daysRemaining` has same UTC parsing bug. Can be off by one day.
- **`Challenges.tsx:86` (confidence: 88)**: `error` state from store never rendered. IndexedDB failures show misleading "No challenges yet" instead of error + retry.
- **`story-e06-s01.spec.ts:12` (confidence: 85)**: `.catch(() => {})` silently swallows `waitForSelector` failure in E2E helper.

### Medium

- **`Challenges.tsx:43` (confidence: 80)**: String interpolation for conditional classes instead of `cn()`. Recurring pattern.
- **`Challenges.tsx:47-48,98,108-109,118` (confidence: 78)**: `h-N w-N` instead of `size-N` shorthand. Recurring.
- **`CreateChallengeDialog.tsx:73-76` (confidence: 72)**: Target accepts fractional numbers for completion/streak types (e.g., 1.5 videos). Add integer validation for non-time types.
- **`Challenges.tsx:89-91` (confidence: 68)**: `loadChallenges` in useEffect fires twice in StrictMode. Add ignore flag.

- **`Challenges.tsx:30` (confidence: 65)**: `ChallengeCard` inline — extract if it grows with delete/edit actions.
