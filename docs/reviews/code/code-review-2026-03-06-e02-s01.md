# Code Review: E02-S01 — Lesson Player Page with Video Playback

**Review Date**: 2026-03-06
**Reviewed By**: Claude Code (code-review agent)

## What Works Well

1. **`useVideoFromHandle` hook design is solid.** Cancellation flag pattern, blob URL cleanup on unmount/handle-change, and permission check/request flow follow the File System Access API correctly.
2. **Proper use of Zustand selectors.** Both `ImportedCourseDetail` and `ImportedLessonPlayer` use individual selectors instead of destructuring the full store.
3. **Lazy loading and route organization.** New routes properly lazy-loaded via `React.lazy()`, separate `/imported-courses/` namespace avoids breaking existing static course system.

## Findings

### High Priority

- **`src/app/pages/ImportedCourseDetail.tsx:25-26` (confidence: 92)**: Dexie queries fire-and-forget with no `.catch()`. Unhandled promise rejection leaves learner stuck on empty page. Fix: Add try/catch with error state.

- **`src/hooks/useVideoFromHandle.ts:31-32` (confidence: 88)**: Catch block maps ALL errors to `'file-not-found'`, losing error discrimination. `SecurityError`, `TypeError`, `AbortError` should be surfaced differently. Fix: Inspect error type, add `console.error` at minimum.

- **`src/app/pages/ImportedLessonPlayer.tsx:37` (confidence: 85)**: `db.importedVideos.get(lessonId)` has no `.catch()`. Unhandled rejection leaves "Loading..." forever. Fix: Add catch handler with error state.

- **`src/db/schema.ts:18` (confidence: 90)**: `EntityTable<VideoProgress, 'courseId'>` is incorrect for compound PK `[courseId+videoId]`. TypeScript allows `db.progress.get(courseId)` (single string) when actual PK is compound. Fix: Use `Table<VideoProgress>` instead.

### Medium

- **`src/hooks/useVideoFromHandle.ts:3` (confidence: 82)**: `error` typed as `string | null` instead of union literal. No TypeScript exhaustiveness checking. Fix: Define `type VideoHandleError = 'file-not-found' | 'permission-denied'`.

- **`src/app/pages/ImportedLessonPlayer.tsx:101-103` (confidence: 78)**: `handleLocateFile` catch silently swallows ALL errors, not just picker cancellation. Fix: Narrow catch to only swallow `AbortError`.

- **`src/app/pages/ImportedCourseDetail.tsx:8-11` (confidence: 75)**: `formatDuration` doesn't handle hours. 2-hour video shows `120:00` instead of `2:00:00`. Fix: Add hour handling.

- **`src/hooks/useVideoFromHandle.ts:6` (confidence: 80)**: Initial state `loading: false` creates intermediate render where nothing shows before effect fires. Fix: Initialize `loading: true` when handle is truthy.

- **Test files (confidence: 72)**: Dexie mocks hardcoded to always succeed. No tests for query rejection paths. Fix: Add Dexie failure test cases.

### Nits

- **`src/app/components/figma/ImportedCourseCard.tsx:82` (confidence: 70)**: `handleCardKeyDown` doesn't call `guardNavigation(e)` — Enter/Space on interactive child still navigates.

- **`src/app/pages/ImportedCourseDetail.tsx:85` (confidence: 68)**: PDFs have `cursor-not-allowed` and `opacity-75` but no tooltip or `aria-disabled`.

- **`src/hooks/useVideoFromHandle.ts:20,22,28` (confidence: 65)**: Non-null assertions (`handle!`) technically unnecessary — could pass handle as parameter to `load()`.
