# Code Review: E02-S04 — PDF Viewer with Page Navigation

**Date**: 2026-02-21
**Reviewer**: Claude Code (adversarial code review agent)
**Story**: E02-S04 — PDF Viewer with Page Navigation

## What Works Well

1. **Solid react-pdf integration.** PdfViewer correctly uses react-pdf's `Document` and `Page` components instead of iframes, with proper TextLayer and AnnotationLayer CSS imports for text selection support. Worker configuration in `src/lib/pdfWorker.ts` follows the recommended Vite pattern.

2. **Strong accessibility foundation.** `role="document"`, `role="toolbar"`, ARIA labels on all icon-only buttons, `aria-expanded`/`aria-haspopup` on the zoom dropdown, and `aria-live="polite"` region for page/zoom announcements. Focus management with `tabIndex={0}` and `focus-visible` ring.

3. **Thoughtful PDF-only lesson handling.** The `primaryPdf` promotion logic in LessonPlayer elegantly handles lessons without video by elevating the first PDF to the primary content area, keeping the remaining PDFs in the Materials tab.

## Findings

### Blockers

- **[2-4-pdf-viewer-with-page-navigation.md:35] AC3 specifies IndexedDB but implementation uses localStorage.** The story states "the last viewed page is restored from IndexedDB" and Task 6 says "Persist and restore last viewed page in IndexedDB". Implementation in `src/lib/progress.ts` uses `localStorage`. While consistent with the existing progress module, it technically fails the AC as written. **Fix**: Update the AC to explicitly say localStorage (acknowledging the existing pattern), or migrate to Dexie.js.

### High Priority

- **[PdfViewer.tsx:24-25] `courseId` and `resourceId` props accepted but never used.** Destructured at line 48 and discarded. Misleads developers into thinking PdfViewer handles its own persistence. **Fix**: Remove from interface and LessonPlayer call sites.

- **[LessonPlayer.tsx:84-94] Debounce timeout not cleaned up on unmount.** If learner navigates away within the 500ms debounce window, `savePdfPage` fires after unmount. **Fix**: `useEffect(() => () => clearTimeout(handlePdfPageChangeRef.current), [])`.

- **[progress.test.ts] No unit tests for `savePdfPage` or `getPdfPage`.** Two new exported functions with zero dedicated unit tests. **Fix**: Add tests covering save, retrieve, new course initialization, overwrite, and `logStudyAction` call.

- **[LessonPlayer.tsx:312-314] Materials tab PDFs save on every page change with no debounce.** Primary PDF uses 500ms debounce, but Materials tab PDFs call `savePdfPage` synchronously on every click. **Fix**: Apply same debounce pattern.

### Medium

- **[PdfViewer.tsx:296-437] `h-X w-X` instead of Tailwind v4 `size-X`.** 20 occurrences. Previously flagged in E01-S03/E01-S04 reviews.

- **[PdfViewer.tsx:366-384] Zoom dropdown no keyboard dismiss.** Pressing Escape while open does nothing. WCAG 2.1 expects popups dismissible via Escape. **Fix**: Add Escape handler or use shadcn/ui DropdownMenu.

- **[PdfViewer.tsx:50-62] 12 separate useState declarations.** Several co-dependent. Could consolidate.

### Nits

- **[PdfViewer.tsx:255]** Error fallback uses `rounded-2xl`, main viewer uses `rounded-[24px]`. Pick one.
- **[PdfViewer.tsx:433]** `window.open(src, '_blank')` — add `'noopener,noreferrer'` for defense-in-depth.
- **[PdfViewer.tsx:109-117]** `goToPage` recreated on every page change due to `currentPage` dependency.
- **[PdfViewer.tsx:136-143]** Page dimensions captured from first page only. Mixed-dimension PDFs will miscalculate fit.
