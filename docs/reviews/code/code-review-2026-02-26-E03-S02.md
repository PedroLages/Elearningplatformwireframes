# Code Review: E03-S02 — Side-by-Side Study Layout

**Date**: 2026-02-26
**Reviewer**: Adversarial Code Review Agent (Opus)
**Branch**: `feature/e03-s02-side-by-side-study-layout`

## What Works Well

1. **Clean responsive architecture.** The `useIsDesktop`/`useIsTablet`/`useIsMobile` hooks provide a clear, testable abstraction over media queries. The three-tier layout (desktop resizable, tablet toggle, mobile stacked+fullscreen) is well-structured.

2. **Controlled tabs with notes-tab gating.** Converting from `defaultValue` to `value`/`onValueChange` controlled tabs was the right call. The logic to hide the Notes tab when the side panel is open prevents confusing duplicate editors on desktop.

3. **Imperative panel API usage.** Using `usePanelRef()` with `expand()`/`collapse()` to sync `notesOpen` state with `react-resizable-panels` is the correct pattern.

## Findings

### Blockers (2)

1. **LessonPlayer.tsx:727-751** — Mobile fullscreen notes overlay has **no Escape key handler** and **no focus trap**. The fixed `z-50` overlay covers the entire screen but keyboard users cannot dismiss with Escape, and Tab can reach elements behind the overlay. WCAG 2.1 AA violation (2.1.2 No Keyboard Trap). Fix: Add `onKeyDown` Escape handler, `role="dialog"`, `aria-modal="true"`, `aria-label`, and a focus-trap effect — or replace with shadcn/ui `Dialog`.

2. **LessonPlayer.tsx:544-555 + 727-751** — When `noteFullScreen` is true, **two NoteEditor instances** are simultaneously mounted (tab + fullscreen overlay). Both receive same `initialContent`/`onSave`, creating a feedback loop via NoteEditor's internal `useEffect` that resets content on `initialContent` change. Fix: Conditionally unmount the tab NoteEditor when fullscreen is active: `!noteFullScreen &&`.

### High Priority (4)

1. **story-e03-s02.spec.ts:119-130** — The "notes available indicator" test is a **no-op**. AC2 explicitly requires indicator behavior but no notes are seeded and no indicator is asserted.

2. **LessonPlayer.tsx:619** — Inline `style={{ overflow: 'visible' }}` on `ResizablePanelGroup` violates Tailwind-only convention. Fix: Use `overflow-visible` class (with `!` important modifier if needed to override library inline styles).

3. **LessonPlayer.tsx:280-288** — `handleNotesToggle` tab fallback chain (`materials > bookmarks > transcript`) can select a non-existent tab for lessons without PDF, video, or captions. Fix: Add `'notes'` as final fallback.

4. **LessonPlayer.tsx:12** — `usePanelRef` imported directly from `react-resizable-panels` bypassing the shadcn/ui wrapper. Fix: Re-export from `resizable.tsx`.

### Medium (3)

1. **LessonPlayer.tsx:647-655** — Close button `h-7 w-7` (28x28px) below WCAG 2.5.8 target size of 44x44px.
2. **LessonPlayer.tsx:309,451,466,467,654,680,689,737** — `h-N w-N` instead of Tailwind v4 `size-N` shorthand throughout.
3. **story-e03-s02.spec.ts:85-97** — Min panel width assertion tolerance (0.30 vs `minSize={35}`) too loose; missing symmetric test for notes panel minimum.

### Nits (3)

1. **LessonPlayer.tsx:4-18** — Import paths mix `../` and `@/` alias inconsistently.
2. **story-e03-s02.spec.ts:155** — Exact match `{ name: 'Notes', exact: true }` in one test vs `/notes/i` regex everywhere else.
3. **LessonPlayer.tsx:573** — Pre-existing inline style from S08 (already migrated to Tailwind class — improvement noted).
