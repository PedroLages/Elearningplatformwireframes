## Findings

### Blockers

#### 1. `fake-indexeddb/auto` breaks jsdom's `localStorage` — 45 tests in `progress.test.ts` fail

**`src/lib/__tests__/progress.test.ts:1,86`**

The `fake-indexeddb/auto` import replaces the global `indexedDB` but clobbers jsdom's `localStorage` API. Every test fails with `TypeError: localStorage.clear is not a function`. This is a shipped regression on `main`.

**Fix**: Replace `/auto` import with explicit `globalThis.indexedDB` assignment.

#### 2. `schema.test.ts` migration test fails — same `localStorage` breakage

**`src/db/__tests__/schema.test.ts:1,289`**

The migration integration test (`should migrate notes from localStorage course-progress`) calls `localStorage.setItem(...)` which throws `TypeError`. The ONE test validating AC2 does not pass.

**Fix**: Same as Finding 1.

### High Priority

#### 3. `migrateBookmarksFromLocalStorage()` races with `db.open()` — tables may not exist

**`src/main.tsx:10-21`**

Both fire in parallel. Bookmark migration calls `db.bookmarks.bulkPut(...)` which triggers auto-open, potentially competing with `db.open()`'s v4 upgrade callback.

**Fix**: Move bookmark migration inside `db.open().then()` chain.

#### 4. `progress.ts` note functions lack `persistWithRetry` — no resilience in production

**`src/lib/progress.ts:222-253`**

`LessonPlayer` imports from `progress.ts` (the production path), not `useNoteStore` (which has retry). The retry logic is dead code.

**Fix**: Wrap `progress.ts` Dexie calls in `persistWithRetry`, or migrate LessonPlayer to use stores in E03-S01.

#### 5. Bookmark migration has zero test coverage (AC3 gap)

**No test file exists**

`migrateBookmarksFromLocalStorage()` in `src/lib/bookmarks.ts:112-133` has no tests.

**Fix**: Add `src/lib/__tests__/bookmarks.test.ts`.

### Medium

#### 6. `useNoteStore` and `useBookmarkStore` are dead code — zero component imports

Both stores are tested (30+ tests) but imported by zero components. Deliberate debt for E03-S01.

#### 7. No migration error path test — AC5 still untested

Error re-throw was added but no test validates the error flow.

#### 8. `CourseCard` progress variant still uses string interpolation for className

**`src/app/components/figma/CourseCard.tsx:526-532`**

The fix was applied to `ProgressCourseCard` but the same pattern in the unified `CourseCard` progress variant was not updated.

### Nits

#### 9. `h-*`/`w-*` used where `size-*` shorthand available

Multiple files in `LessonPlayer.tsx` and `CourseCard.tsx`.

#### 10. `handleNoteChange` ignores async errors from `saveNote`

**`src/app/pages/LessonPlayer.tsx:242-246`**

Fire-and-forget with no error handling. Will be resolved when LessonPlayer migrates to stores.
